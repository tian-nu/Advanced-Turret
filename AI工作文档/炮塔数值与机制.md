## 🔫 炮塔详细机制

### 1. 机枪炮塔 (Machine Gun Turret)

**定位**: 入门级高射速炮塔

**常量定义**:
```java
public static final int FIRE_RATE = 5;           // 射击间隔 (tick)
public static final double SEARCH_RADIUS = 32.0; // 搜索范围
public static final double BULLET_SPEED = 3.0;   // 子弹速度
public static final float BULLET_DAMAGE = 4.0F;  // 子弹伤害
public static final int ENERGY_COST = 100;       // 能量消耗/发
```

**子弹类型**: `TurretBulletEntity`
- 直线飞行
- 击中实体/方块后销毁
- 支持无敌帧绕过

**弹药**: `MACHINE_GUN_BULLET`

**特殊机制**: 无

---

### 2. 磁轨炮塔 (Railgun Turret)

**定位**: 高伤害穿透型炮塔

**常量定义**:
```java
public static final int FIRE_RATE = 100;          // 射击间隔 (tick, 5秒)
public static final double SEARCH_RADIUS = 64.0;  // 搜索范围
public static final double BULLET_SPEED = 6.0;    // 子弹速度 (高速)
public static final float BULLET_DAMAGE = 60.0F;  // 子弹伤害
public static final int ENERGY_COST = 10000;      // 能量消耗/发
public static final int PENETRATION_COUNT = 3;    // 穿透次数
```

**子弹类型**: `RailgunBulletEntity`
- 高速飞行 (速度6.0)
- 穿透最多3个目标
- **穿透无衰减**: 每次穿透伤害不变
- 前3tick忽略方块碰撞
- 分离碰撞检测（实体优先）

**弹药**: 无需弹药

**特殊机制**:
- 穿透: 击中目标后不销毁，继续飞行
- 穿透无衰减: 伤害不会随穿透次数减少
- 已击中实体ID记录，防止重复伤害
- 电弧粒子效果

---

### 3. 火箭炮塔 (Rocket Turret)

**定位**: 远程爆炸型炮塔

**常量定义**:
```java
public static final int FIRE_RATE = 100;             // 射击间隔 (tick, 5秒)
public static final double SEARCH_RADIUS = 48.0;     // 搜索范围
public static final double BULLET_SPEED = 2.0;       // 子弹初始速度
public static final double ACCELERATION = 0.15;      // 加速度 (初始2速，20tick后达到5速)
public static final float DIRECT_DAMAGE = 10.0F;     // 直击伤害
public static final float EXPLOSION_DAMAGE = 10.0F;  // 爆炸伤害
public static final float EXPLOSION_RADIUS = 4.0F;   // 爆炸半径
public static final int ENERGY_COST = 5000;          // 能量消耗/发
```

**子弹类型**: `RocketEntity`
- 直线飞行
- **越飞越快**: 初始速度2.0，每tick增加0.15，20tick(1秒)后达到速度5.0
- 击中实体/方块后爆炸
- 范围伤害随距离衰减

**弹药**: `ROCKET`

**特殊机制**:
- 直击伤害: 直接命中目标
- 爆炸伤害: 范围AOE，伤害 = `explosionDamage * (1 - distance / radius)`
- 破坏插件: 装备后可破坏方块
- 烟雾粒子尾迹

## 🎯 子弹系统

### 子弹继承关系

```
TurretProjectileEntity (抽象基类)
├── TurretBulletEntity     (机枪子弹 - 击中销毁)
├── RailgunBulletEntity    (磁轨炮 - 穿透)
├── RocketEntity           (火箭弹 - 爆炸)
├── MissileEntity          (导弹 - 追踪+爆炸)
├── GrenadeEntity          (榴弹 - 抛物线+爆炸)
├── LaserBeamEntity        (激光 - 持续伤害)
└── JunkProjectileEntity   (垃圾 - 随机材质)
```

### 基类属性 (TurretProjectileEntity)

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `damage` | float | - | 子弹伤害 |
| `lifetime` | int | 100 | 生命周期(tick) |
| `sourcePos` | BlockPos | - | 炮塔位置 |
| `basePos` | BlockPos | - | 基座位置 |
| `ignoreSourceBlockTicks` | int | 2 | 跳过炮塔碰撞tick数 |

### 碰撞检测机制

**分离碰撞检测** (所有子弹必须实现):
```java
@Override
public void tick() {
    // 1. 先检测实体碰撞
    EntityHitResult entityHit = ProjectileUtil.getEntityHitResult(
        level, this, currentPos, nextPos, 
        boundingBox.expandTowards(movement).inflate(1.0),
        this::canHitEntity
    );
    
    if (entityHit != null) {
        this.onHit(entityHit);
        return;
    }
    
    // 2. 再检测方块碰撞
    BlockHitResult blockHit = level.clip(new ClipContext(...));
    
    // 3. 特殊处理：基座销毁、炮塔跳过
}
```

### 无敌帧绕过

```java
protected void clearInvulnerability(LivingEntity entity) {
    entity.invulnerableTime = 0;
    entity.hurtTime = 0;
}
```

## 🔧 升级组件效果

### 组件效果表

| 组件 | 每层效果 | 最大效果 | 计算公式 |
|------|----------|----------|----------|
| **攻击增强** | +10%伤害 | +200% (3层) | `damage * min(3.0, 1.0 + count * 0.10)` |
| **射程扩展** | +1格 | +3格 | `min(32.0, baseRange + count)` |
| **射速超频** | -5%冷却 | -15% (3层) | `max(1, baseRate * max(0.20, 1.0 - count * 0.05))` |
| **能效优化** | -5%能耗 | -15% (3层) | `max(1, baseCost * max(0.20, 1.0 - count * 0.05))` |
| **精度提升** | 减少散射 | 待定 | 待实现 |

### 组件叠加规则

- 每槽最多堆叠4个同类型组件
- 按面独立管理（6面各一套）
- 只有该面有炮塔时槽位可见

## 🎯 目标识别

### 目标分类

| 分类 | 检测方式 | 包含类型 |
|------|----------|----------|
| **敌对** | `instanceof Enemy` | 僵尸、骷髅、爬行者等 |
| **中立** | `instanceof NeutralMob` | 末影人、猪灵、狼等 |
| **友善** | `Animal/AmbientCreature/WaterAnimal` | 动物、蝙蝠、鱼类 |
| **玩家** | `instanceof Player` | 玩家（非创造/观察者） |

### 目标选择流程

```
1. 黑名单检查 → 强制攻击
2. 白名单检查 → 强制跳过
3. 目标模式匹配 (flags位运算)
4. 友伤保护检查 (主人/驯服生物)
5. 范围检查
6. 视线检测 (三点：头/身/脚)
7. 选择最近目标
```
